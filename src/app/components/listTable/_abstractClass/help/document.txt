
Title: AbstractList Component - Angular Documentation

1. **Introduction**
   - The AbstractList component serves as a base class for building dynamic, reusable list interfaces in Angular applications.
   - It integrates powerful features such as responsive table layouts, form management, and advanced CRUD operations.
   - By extending this class, developers can focus on domain-specific logic while leveraging common functionalities.

2. **Key Features**
   - **Responsive Design**:
     - Dynamically adjusts table columns based on screen size using custom breakpoints.
     - Ensures optimal viewing experience across devices (mobile, tablet, desktop).
   - **CRUD Operations**:
     - Simplifies fetching, creating, updating, and deleting data with integrated methods.
     - Works seamlessly with services like `ApiMcrServiceInterface` for data management.
   - **Two-Way Binding**:
     - Enables synchronization between form controls and data models, reducing boilerplate code.
   - **Paginator Integration**:
     - Utilizes Angular Material's `MatPaginator` for easy data pagination.
   - **Breakpoints Subscription**:
     - Observes screen size changes and adjusts the displayed columns dynamically.
   - **Form Management**:
     - Abstracts form initialization, validation, and reset logic, making it easier to handle user inputs.
   - **User Permissions**:
     - Provides built-in checks for user-specific access and edit permissions.
   - **Error Handling**:
     - Integrates `ToastrService` for user-friendly error notifications.

3. **Dependencies**
   - **Angular Core Modules**:
     - Core components like `Component`, `FormGroup`, `ChangeDetectorRef`.
   - **Angular Material Components**:
     - `MatTable`, `MatPaginator`, `MatDialog` for UI functionalities.
   - **CDK Layout**:
     - Used for breakpoint detection and responsive design.
   - **RxJS**:
     - Enables reactive programming with features like `Subject`, `Subscription`, and `debounceTime`.
   - **Lodash**:
     - Simplifies deep equality checks using `isEqual`.

4. **Abstract Methods**
   - Developers must implement these methods when extending AbstractList:
     - **columnsToDisplay**: Defines columns for different screen sizes.
     - **form: FormGroup**: Specifies the reactive form structure.
     - **aliasService**: Provides the service for CRUD operations.
     - **getInitForm()**: Returns the initial form group with controls and validators.
     - **getClassName()**: Supplies the entity name being managed (e.g., `Contact`, `Product`).
     - **formatDataToInput(element: T)**: Prepares data for display or editing.
     - **convertColumnToStr(column: string)**: Converts column data to string for display purposes.
     - **_formalizer(element: T)**: Ensures data is properly formatted before submission.

5. **Lifecycle Hooks**
   - **ngOnInit**:
     - Initializes subscriptions for form and route data.
     - Sets up user permissions and loads initial table data.
   - **ngAfterViewInit**:
     - Configures paginator subscriptions for handling page changes dynamically.
   - **ngOnDestroy**:
     - Unsubscribes from all active subscriptions to prevent memory leaks.

6. **Public and Protected Methods**
   - **Public**:
     - `toggleExpand(element: T)`: Toggles row expansion to display detailed data.
   - **Protected**:
     - `_observeBreakpoints`: Dynamically adjusts columns based on screen size.
     - `_editMode`: Enables edit mode for selected rows.
     - `_viewMode`: Disables edit mode and switches to view mode.
     - `_enableEditControls`: Activates form controls for editing.
     - `_disableEditControls(excepts?: string[])`: Deactivates form controls, except specified ones.
     - `_enable2wayBinding`: Sets up two-way binding between form controls and row data.
     - `_disable2wayBinding`: Cleans up value change subscriptions.

7. **Usage Example**
   - Demonstrates how to create a concrete implementation of AbstractList:
     ```typescript
     @Component({
       selector: 'app-contact-list',
       templateUrl: './contact-list.component.html',
       styleUrls: ['./contact-list.component.scss']
     })
     export class ContactListComponent extends AbstractList<Contact> {
       columnsToDisplay = ['name', 'email', 'phone'];
       columnsToDisplay_LargeScreen = ['name', 'email', 'phone', 'address'];
       columnsToDisplay_SmallScreen = ['name', 'email'];
       columnsToDisplay_MiddleScreen = ['name', 'email', 'phone'];

       form: FormGroup;

       aliasService: ContactService;

       constructor(
         toastrService: ToastrService,
         parseUtilsService: ParseUtilsService,
         changeDetectorRef: ChangeDetectorRef,
         editStateSrv: EditStateService,
         authService: AuthService,
         breakpointObserver: BreakpointObserver,
         dialog: MatDialog,
         academieAuditeService: AcademieAuditeService,
         route: ActivatedRoute,
         contactService: ContactService
       ) {
         super(
           toastrService,
           parseUtilsService,
           changeDetectorRef,
           editStateSrv,
           authService,
           breakpointObserver,
           dialog,
           academieAuditeService,
           route
         );
         this.aliasService = contactService;
         this.form = this.getInitForm();
       }

       getInitForm(): FormGroup {
         return new FormGroup({
           name: new FormControl('', Validators.required),
           email: new FormControl('', [Validators.required, Validators.email]),
           phone: new FormControl(''),
           address: new FormControl('')
         });
       }

       getClassName(): string {
         return 'Contact';
       }

       async formatDataToInput(element: Contact): Promise<void> {
         element.phone = element.phone?.replace(/-/g, ' ');
       }

       convertColumnToStr(column: string): string {
         return column.toUpperCase();
       }

       _formalizer(element: Contact): void {
         element.phone = element.phone?.replace(/\s/g, '-');
       }
     }
     ```
   - This example implements a `ContactListComponent` that manages a list of contacts.

8. **Conclusion**
   - The AbstractList component simplifies the development of list-based UIs.
   - It promotes code reusability and adheres to Angular's best practices.
   - By extending this class, developers can focus on application-specific logic while leveraging common functionality.
